Secure SSL Certificate Validation

[This documentation is for preview only, and is subject to change in later releases. Blank topics are included as placeholders.]

Secure SSL validation reduces the chance that another process will discover the credentials you use to access a SQL Server. For more information, see Encrypting Connections to SQL Server.

Details

When trustServerCertificate is false and encrypt is true, Microsoft JDBC Driver 4.0 for SQL Server requires that the name (or IP address) in a Subject Common Name (CN) or Subject Alternative Name (SAN) in a SQL Server SSL certificate should exactly match the server name (or IP address) specified in the connection string or hostNameInCertificate connection property. For more information, see the description of encrypt in Setting the Connection Properties.

If this change causes an existing application to no longer connect, you can fix the application using one of the following:

  • Ensure that the server name (or serverName property) in the connection string exactly matches the CN or SAN in SQL Server SSL certificate.

  • If the server name cannot be modified, hostNameInCertificate connection property should be initialized to the CN or SAN in SQL Server SSL certificate.

  • If the connection string parameters cannot be modified, issue a certificate that specifies a CN or SAN that is the same as the server name or hostNameInCertificate.

    If the server name in the SQL Server SSL certificate contains a wildcard character, specify the hostNameInCertificate property and pass the server name, including wildcard character that is used in the SQL Server SSL certificate.

Internationalized domain names (IDN) are not supported with secure SSL certificate validation.
