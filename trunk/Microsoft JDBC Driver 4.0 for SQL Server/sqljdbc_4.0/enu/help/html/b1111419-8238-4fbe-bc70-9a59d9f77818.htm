Extended Protection

[This documentation is for preview only, and is subject to change in later releases. Blank topics are included as placeholders.]

Beginning in Microsoft JDBC Driver 4.0 for SQL Server, the JDBC driver supports service binding in Extended Protection. Channel binding is not currently supported in the JDBC driver. For more information about Extended Protection in SQL Server, see Connecting to the Database Engine Using Extended Protection.

Details

A JDBC driver application must use Windows Authentication (integratedSecurity connection property) to allow a server to provide Extended Protection.

A connection will fail if:

  • A JDBC driver application connects to a server that has Extended Protection set to Required and the SPN sent by the client does not match the SPN of the SQL Server service.

  • A JDBC driver application running on a protected operating system connects to a server that has Extended Protection set to Allowed and the SPN sent by the client does not match the SPN of the SQL Server service.

  • Extended Protection is set to Required or Allowed and a JDBC driver application uses Encrypt = true.
